# OpenCode Enterprise

OpenCode offers enterprise-grade features for team deployment, security, and administration in organizational settings.

## Enterprise Features

- Team collaboration and sharing capabilities
- Centralized configuration management
- Enhanced security and compliance options
- Administrative controls and monitoring
- Integration with enterprise identity providers
- On-premises deployment options

## Team Deployment

For deploying OpenCode across teams or organizations:

1. Standardized installation procedures
2. Shared configuration templates
3. Centralized API key management (where appropriate)
4. Consistent environment setup
5. Version control integration practices

## Security Considerations

When using OpenCode in enterprise environments:

- Secure handling of API keys and credentials
- Network security and proxy configurations
- Data privacy and compliance considerations
- Access controls and permissions
- Audit logging and monitoring capabilities

## Configuration Management

Enterprise deployment benefits from:

- Centralized configuration files
- Standardized .opencode/skills directories
- Pre-approved skill sets for teams
- Consistent agent behavior across developers
- Template AGENTS.md files for project initialization

## Administration

Administrative capabilities may include:

- User management and access controls
- Usage monitoring and reporting
- Policy enforcement and compliance checking
- Integration with existing identity systems
- Custom skill distribution and approval workflows

## Getting Started with Enterprise Deployment

Refer to the [Enterprise](/docs/enterprise/) documentation for specific guidance on deploying OpenCode in enterprise environments, including detailed installation instructions, configuration options, and best practices for team usage.
